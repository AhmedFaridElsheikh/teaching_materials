Math library

Math functions you already know — now in Java

Java's Math class gives you the same functions from your calculator and MATLAB. You know these from solving circuits.

public class CircuitMath {
    public static void main(String[] args) {

        // Impedance magnitude: |Z| = sqrt(R² + (wL - 1/wC)²)
        double R  = 100.0;   // Ohms
        double L  = 0.1;    // Henry
        double C  = 100e-6; // Farad
        double f  = 50.0;   // Hz (grid frequency!)
        double w  = 2 * Math.PI * f;

        double XL  = w * L;
        double XC  = 1.0 / (w * C);
        double Zmag = Math.sqrt(R*R + Math.pow(XL - XC, 2));

        // Phase angle: theta = atan2(X, R)
        double theta_rad = Math.atan2(XL - XC, R);
        double theta_deg = Math.toDegrees(theta_rad);

        System.out.println("XL = " + XL);
        System.out.println("XC = " + XC);
        System.out.println("|Z| = " + Zmag + " Ohms");
        System.out.println("theta = " + theta_deg + " degrees");

        // Exponential decay — you'll see this again in RC!
        double tau = R * C;
        double t   = 0.02; // 20 ms
        double vC  = 100 * Math.exp(-t / tau);
        System.out.println("vC(20ms) = " + vC + " V");
    }
}
Why you'll use this: Load flow, protection relay calculations, power factor correction — all involve complex impedance, angles, and exponentials. Java runs inside PLCs and SCADA systems at every substation.

int    bad  = 1/2;   // = 0 !
double good = 1.0/2; // = 0.5

Source-free RC: from equation to code

You already solved this analytically. Now let's compute it — and verify our formula.

v(t) = V₀ · e^(−t/τ), τ = RC

public class RCSourceFree {
    public static void main(String[] args) {

        double R   = 1000.0;  // 1 kΩ
        double C   = 100e-6; // 100 µF
        double V0  = 12.0;   // initial voltage (V)
        double tau = R * C;   // τ = RC = 0.1 s

        System.out.println("tau = " + tau + " s");

        // Evaluate at specific time points
        double[] times = {0.0, 0.1, 0.2, 0.5, 1.0};

        System.out.println("t(s)   v(t) V");
        for (int i = 0; i < times.length; i++) {
            double t = times[i];
            double v = V0 * Math.exp(-t / tau);
            System.out.printf("%.2f   %.4f%n", t, v);
        }

        // Find time to reach 5% of V0
        double t5pct = -tau * Math.log(0.05);
        System.out.println("5% reached at t = " + t5pct + " s");
        // This is ≈ 3τ — matches your circuit theory!
    }
}
Expected output:
tau = 0.1 s
t=0.0 → 12.0000 V
t=0.1 → 4.4147 V  (= V₀·e⁻¹ = 0.368·V₀)
t=0.2 → 1.6240 V
t=0.5 → 0.0810 V
5% reached at t = 0.2996 s ≈ 3τ ✓
The for loop over an array of time points is the exact same idea as MATLAB's time vector. In protection engineering, you compute transient voltages at specific instants to set relay pickup times.

Solving a linear system — nodal analysis

Nodal analysis gives you Ax = b. Here's a simple 2-node circuit solved in Java using Cramer's rule (easy to code, you already know the formula).

[ G₁₁ G₁₂ ] [ V₁ ] [ I₁ ]
[ G₂₁ G₂₂ ] [ V₂ ] = [ I₂ ]

public class NodalAnalysis {

    // Solve 2x2 system using Cramer's rule
    static double[] solve2x2(double[][] A, double[] b) {
        double det = A[0][0]*A[1][1] - A[0][1]*A[1][0];
        double x1  = (b[0]*A[1][1] - A[0][1]*b[1]) / det;
        double x2  = (A[0][0]*b[1] - b[0]*A[1][0]) / det;
        return new double[] {x1, x2};
    }

    public static void main(String[] args) {
        // R1=2Ω, R2=4Ω, R3=4Ω, Is1=5A, Is2=2A
        double[][] G = {
            { 0.75, -0.25},
            {-0.25,  0.50}
        };
        double[] I  = {5.0, 2.0};

        double[] V = solve2x2(G, I);
        System.out.printf("V1 = %.4f V%n", V[0]);
        System.out.printf("V2 = %.4f V%n", V[1]);
    }
}
Power flow (the core of power systems operation) is exactly this — a large Ax = b system solved thousands of times per second. Tools like PSS/E and OpenDSS use this under the hood. You'll code something like this in a Power Systems Analysis course.

Generating a step function u(t)

The unit step u(t - t₀) is fundamental in circuit analysis. Here's how to generate it, and how to use it to model a switched source.

public class StepFunction {

    // Unit step: returns 1.0 if t >= t0, else 0.0
    static double step(double t, double t0) {
        return (t >= t0) ? 1.0 : 0.0;
    }

    // Rectangular pulse: on from t1 to t2
    static double pulse(double t, double t1, double t2) {
        return step(t, t1) - step(t, t2);
    }

    public static void main(String[] args) {

        // RC circuit: Vs switches on at t=0.05s
        double R   = 1000.0;
        double C   = 100e-6;
        double tau = R * C;
        double Vs  = 10.0;   // source voltage
        double t0  = 0.05;  // switch closes at 50ms

        double[] times = {0.0, 0.04, 0.05, 0.1, 0.15, 0.3};

        System.out.println("t(s)    Vs(t)   vC(t)");
        for (int i = 0; i < times.length; i++) {
            double t   = times[i];
            double vsrc = Vs * step(t, t0);
            double vC;
            if (t < t0) {
                vC = 0.0;             // before switch
            } else {
                vC = Vs * (1 - Math.exp(-(t-t0)/tau));
            }
            System.out.printf("%.3f   %.1f    %.4f%n", t, vsrc, vC);
        }

        // Pulse example: fault lasts from t=0.1 to t=0.2s
        System.out.println("\nFault signal:");
        double[] tt = {0.05, 0.1, 0.15, 0.2, 0.25};
        for (int i = 0; i < tt.length; i++) {
            System.out.printf("t=%.2f  fault=%s%n",
                tt[i], pulse(tt[i],0.1,0.2)==1.0 ? "ON" : "off");
        }
    }
}
Fault modeling in power systems is exactly this — a step or pulse disturbance. Relay engineers code these waveforms to test protective schemes in simulation. The ternary operator ? : is just a compact if-else.
Arrays + for loops

State-space + Forward Euler — the big picture

This is the motivation for learning arrays and for loops. A 1st-order RC in state-space: state variable is vC(t), input is Vs.

dvC/dt = −vC/(RC) + Vs/(RC)  →  x' = Ax + Bu

public class ForwardEulerRC {
    public static void main(String[] args) {

        // Circuit parameters
        double R   = 1000.0;
        double C   = 100e-6;
        double tau = R * C;   // 0.1 s
        double Vs  = 10.0;   // step input voltage

        // Simulation settings
        double dt   = 0.001;  // time step 1 ms  (dt << tau!)
        double tEnd = 0.5;    // simulate 500 ms
        int    N    = (int)(tEnd / dt) + 1;

        // Allocate arrays to store the full trajectory
        double[] t  = new double[N]; // time vector
        double[] vC = new double[N]; // state: capacitor voltage

        // Initial condition
        t[0]  = 0.0;
        vC[0] = 0.0; // capacitor starts uncharged

        // Forward Euler loop: x[k+1] = x[k] + dt * f(x[k], u)
        for (int k = 0; k < N-1; k++) {
            t[k+1] = t[k] + dt;

            // State equation: dvC/dt = (-vC + Vs) / tau
            double dvC_dt = (-vC[k] + Vs) / tau;

            // Euler step
            vC[k+1] = vC[k] + dt * dvC_dt;
        }

        // Compare numerical vs analytical at a few points
        System.out.println("t(s)    Euler     Exact    Error%");
        int[] check = {0, 100, 200, 300, 499};
        for (int i = 0; i < check.length; i++) {
            int    k    = check[i];
            double exact = Vs * (1 - Math.exp(-t[k]/tau));
            double err  = Math.abs(vC[k]-exact) / Vs * 100;
            System.out.printf("%.3f   %.4f    %.4f   %.3f%n",
                               t[k], vC[k], exact, err);
        }
    }
}
t=0.000 Euler=0.0000 Exact=0.0000 Error=0.000%
t=0.100 Euler=6.3194 Exact=6.3212 Error=0.018%
t=0.200 Euler=8.6401 Exact=8.6466 Error=0.065%
t=0.499 Euler=9.9296 Exact=9.9329 Error=0.033%

Small step → small error. This is numerical integration!
Why does this matter? EMTP (ElectroMagnetic Transients Program) — the industry standard for simulating power systems transients (lightning, switching, fault currents) — is literally this algorithm applied to every branch in the network, thousands of nodes, with dt = 50 µs. Power electronics simulation (inverters, converters) uses the same idea. Understanding this loop is understanding how the grid is simulated.
